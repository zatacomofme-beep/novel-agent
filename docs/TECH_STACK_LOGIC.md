### 2.4 因果图谱建立流程（Neo4j）

```
章节生成完成
        │
        ▼
┌──────────────────────────────────────────────────────────────────┐
│  CausalGraphService                                               │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ Step 1: 事件抽取 (Event Extraction)                          │ │
│  │ - AI 分析章节内容                                            │ │
│  │ - 识别关键事件：冲突/决定/发现/转折                          │ │
│  │ - 输出：List[Event] {name, chapter, type, participants[]}    │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                              │                                   │
│                              ▼                                   │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ Step 2: 因果关系抽取 (Causal Extraction)                     │ │
│  │ - AI 分析事件间因果关系                                      │ │
│  │ - 判断类型：direct / indirect / implied                     │ │
│  │ - 评估置信度：0.0 ~ 1.0                                     │ │
│  │ - 输出：List[CausalLink] {cause, effect, type, confidence}   │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                              │                                   │
│                              ▼                                   │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ Step 3: 写入 Neo4j                                           │ │
│  │                                                             │ │
│  │   MERGE (e1:Event {name: "密信出现", chapter: 19})          │ │
│  │   MERGE (e2:Event {name: "真相大白", chapter: 50})          │ │
│  │   MERGE (e1)-[:CAUSED {type: "indirect", confidence: 0.87}] │ │
│  │            ->(e2)                                            │ │
│  │                                                             │ │
│  │   MERGE (c:Character {name: "林舟"})                        │ │
│  │   MERGE (e1)-[:INVOLVES]->(c)                               │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                              │                                   │
│                              ▼                                   │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ Step 4: 伏笔追踪关联 (Foreshadowing Link)                   │ │
│  │ - 匹配 Open Threads 数据库                                  │ │
│  │ - 建立 :LEADS_TO 关系                                       │ │
│  │   MERGE (f:Foreshadow {id: "open_thread_001"})              │ │
│  │   MERGE (e:Event {name: "真相大白"})                       │ │
│  │   MERGE (f)-[:LEADS_TO {chapter: 50}]->(e)                 │ │
│  └─────────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────────────────────────────────┐
│  因果溯源查询示例                                                 │
│                                                                  │
│  // 从第50章往前溯源10层因果链                                     │
│  MATCH path = (end:Event {chapter: 50})<-[:CAUSED*1..10]-(start)│
│  RETURN path ORDER BY length(path) DESC LIMIT 5                   │
│                                                                  │
│  // 计算角色因果影响力                                            │
│  CALL gds.pageRank.stream('causalGraph')                         │
│  YIELD nodeId, score                                             │
│  RETURN gds.util.asNode(nodeId).name AS character, score         │
│  ORDER BY score DESC                                             │
│                                                                  │
│  // 检测伏笔回收状态                                              │
│  MATCH (f:Foreshadow)-[r:LEADS_TO]->(e:Event)                   │
│  WHERE f.status = 'open'                                        │
│  RETURN f, r, e ORDER BY r.chapter                              │
└──────────────────────────────────────────────────────────────────┘
```
