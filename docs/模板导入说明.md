### 3. 文件结构

```
backend/
├── data/
│   └── templates_24.json          # 23 套模板的 JSON 数据文件
├── services/
│   └── prompt_template_service.py # 从 JSON 文件动态加载模板
└── scripts/
    ├── generate_templates_json.py # 生成 JSON 文件的脚本
    ├── import_templates.py        # 手动导入数据库的脚本
    ├── update_service_templates.py # 更新服务中的模板
    └── clean_service_file.py      # 清理服务文件
```

### 4. 自动导入机制

当用户首次访问模板库页面时，后端会自动执行以下操作：

1. 检查数据库中是否有系统模板
2. 如果没有，从 `SYSTEM_TEMPLATES` 读取所有模板
3. 批量插入到 `prompt_templates` 表
4. 后续访问直接使用数据库中的数据

### 5. 如何查看模板

访问前端页面：
```
http://localhost:3000/dashboard/prompt-templates
```

### 6. 模板特点

- ✅ **非硬编码**: 数据存储在独立的 JSON 文件中
- ✅ **易于维护**: 修改模板只需编辑 JSON 文件
- ✅ **自动导入**: 首次访问自动导入数据库
- ✅ **分类清晰**: 10 个大类，23 套模板
- ✅ **实用性强**: 每套模板都有详细的使用说明和示例

### 7. 如何添加新模板

1. 编辑 `/backend/data/templates_24.json` 文件
2. 在数组末尾添加新的模板对象
3. 重启后端服务
4. 访问模板库页面，新模板会自动导入

### 8. 模板数据结构

每个模板包含以下字段：
```json
{
  "name": "模板名称",
  "tagline": "一句话简介",
  "description": "详细描述",
  "category": "所属分类",
  "sub_category": "子分类",
  "tags": ["标签 1", "标签 2"],
  "difficulty_level": "beginner|intermediate|advanced",
  "recommended_scenes": ["场景 1", "场景 2"],
  "variables": [
    {
      "name": "变量名",
      "description": "变量说明",
      "required": true
    }
  ],
  "content": "模板内容（支持 Markdown）"
}
```
