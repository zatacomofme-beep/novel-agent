# 长篇小说创作智能体：技术架构与 100+ 技术细节优化路线图

本文件详细规划了 `novel-agent` 从当前架构向“产品级”应用进化的技术路径，目标是实现 100+ 个可量化的技术细节优化。

## 1. 核心架构升级：从线性到多维
*   **多智能体编排 (Orchestration)**: 在现有的 LangGraph 基础上，引入 **Sub-graph 递归架构**。将“章节创作”拆分为独立的子图，由专用的 `Micro-Agents`（如描写、对话、逻辑、伏笔）进行更细粒度的控制。
*   **状态机增强 (State Management)**: 引入 **Undo/Redo 状态回溯**。当人类编辑已生成内容时，系统自动通过 `State Reconstructor` 同步状态，确保全书逻辑链条不断裂。

## 2. 知识与一致性引擎：GraphRAG 深度整合
*   **因果图谱 (Causal Graph)**: 放弃纯向量搜索，引入图数据库（如 Neo4j）。存储 `事件 A -> 导致 -> 事件 B` 的因果链条。
*   **动态上下文压缩 (Context Compression)**: 自动识别已完成情节中的“非核心细节”并进行语义压缩，仅保留关键转折点（Key Points）在 Prompt 中，以应对数百万字的上下文窗口限制。
*   **多级内存架构**:
    *   L1 (工作内存): 当前场景详细 Prompt。
    *   L2 (短期内存): 最近 3-5 章的滚动摘要。
    *   L3 (长期内存): 基于知识图谱的世界观、人物卡设定。

## 3. 智能模型路由 (Smart Routing 2.0)
*   **基于任务复杂度的分流**:
    *   **低耗路由 (Low-cost)**: 背景渲染、环境描写调用轻量化模型。
    *   **高精路由 (High-precision)**: 伏笔回收、情节冲突点、大纲重构强制调用 Claude 3.5 Sonnet。
*   **Token 熔断机制**: 实时监控每章生成的 Token 消耗，对异常循环生成进行自动拦截。

## 4. 自动化评测与内容安全
*   **语义冲突自动检测**: 利用 LLM 定期扫描生成的文本与 `Story Bible` 之间的语义偏差值。
*   **风格迁移与指纹识别**: 确保每一章的语言风格（如“鲁迅风”、“玄幻快节奏”）的一致性评分在 0.8 以上。
