# 主链 Smoke 清单

基于 2026-03-26 当前仓库状态整理。

这份文档只做一件事：
把当前产品最重要的人工冒烟链路固定下来，后续每次较大改动后都按这份清单回归。

---

## 使用规则

- 按顺序执行，不跳步骤。
- 先跑 `Smoke 01`，再跑 `Smoke 02`，最后跑 `Smoke 03`。
- 任一条主链不通过，都要先回写 [implementation-task-table.md](/Users/libenshi/Desktop/novels/docs/architecture/implementation-task-table.md) 再继续开发。
- 这份清单默认在 `dashboard -> story-room` 的单工作台路径下执行，不再回旧页面找能力。

## Smoke 01: 单人主链

目标：
验证 `创建项目 -> 三级大纲 -> 正文 -> 终稿 -> 设定沉淀 -> 下一章` 是否完整可用。

### 步骤

1. 进入首页，点击 `创建新书`，完成登录或注册。
2. 在 `dashboard` 创建一本新书，填写：
   - 书名
   - 总字数
   - 单章目标字数
   - 小说类别
   - 气质
3. 确认创建后，页面应直接进入该书的 `story-room`，默认落在 `三级大纲`。
4. 输入脑洞或上传大纲素材，生成三级大纲。
5. 在大纲区手动修改任意一级、二级、三级节点并保存。
6. 进入正文区，确认顶部主按钮只给出当前明确动作。
7. 选定某条三级大纲，生成第一版正文。
8. 在正文中手动补写一段，再保存本章。
9. 点击 `检查并收口`，等待终稿结果。
10. 在终稿区查看：
    - 原稿 / 优化稿对比
    - 章节总结
    - 设定建议
11. 处理至少 1 条设定建议。
12. 点击 `确认并去下一章`。
13. 确认已自动进入下一章正文区。

### 预期结果

- 创建新书后不会回到模糊入口，而是直接进入该书工作台。
- 三级大纲可生成、可改、可继续作为正文起稿依据。
- 正文区主按钮始终清晰，不会同时出现多个语义重叠的主动作。
- 终稿区不仅展示结果，还能继续确认设定、确认终稿、进入下一章。
- 进入下一章后，系统会继续带入上一章总结和已确认设定。

## Smoke 02: 多卷 / 分线链路

目标：
验证 `story-room` 内的卷、分线、章节作用域切换不会破坏主链。

### 步骤

1. 打开一本已有大纲和正文的书。
2. 在 `story-room` 创建一个新分线。
3. 在新分线下创建或切换到一个新卷。
4. 确认三级大纲、正文区、设定区都跟着当前分线 / 卷切换。
5. 在当前分线的某一章生成正文并保存。
6. 在设定区新增 1 条人物或物品设定。
7. 切回默认主线，确认默认主线不被错误覆盖。
8. 再切回新分线，确认刚才的章节和设定仍能看到。

### 预期结果

- 分线 / 卷切换后，当前作用域清晰可见。
- 不同分线下的章节和设定不会互相串线。
- 下一章候选、正文生成、大纲挂接都跟随当前分线 / 卷。

## Smoke 03: 本机稿恢复 / 断网保护

目标：
验证正文区的本机保稿和恢复链路可用。

### 步骤

1. 打开任意一章正文。
2. 手动输入一段明显的新内容，但先不要保存。
3. 等待本机暂存提示出现。
4. 直接刷新页面，或切到别章后再切回来。
5. 确认系统出现恢复提示，并可恢复刚才的内容。
6. 模拟断网或关闭后端请求，再继续编辑正文。
7. 确认正文区仍可继续输入，并显示离线保稿状态。
8. 恢复联网后，确认内容仍可保存为正式章节。

### 预期结果

- 本机暂存能正确记录未保存正文。
- 刷新或切章后能识别并恢复对应草稿。
- 离线时不会阻塞写作。
- 重新联网后可以继续保存和收口，不出现整章丢失。

## 执行后回写

每次 smoke 执行后，至少记录三件事：

- 哪一条 smoke 执行通过
- 卡在哪一步
- 如果失败，对应回写到任务表里的哪个任务

如果三条 smoke 都通过，才可以认为当前主链处于“可继续迭代”状态。
