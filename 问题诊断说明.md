### 可能的问题原因

#### 1. 用户未登录
- 前端 `apiFetchWithAuth` 函数需要有效的 JWT token
- 如果用户未登录或 token 过期，会导致请求失败
- **检查方法**：打开浏览器控制台，查看是否有认证错误

#### 2. 跨域问题 (CORS)
- 如果后端没有正确配置 CORS，前端请求会被浏览器拦截
- **检查方法**：打开浏览器控制台，查看是否有 CORS 错误

#### 3. 前端 API 调用失败
- 网络请求可能失败（后端服务未启动、端口被占用等）
- **检查方法**：打开浏览器 Network 面板，查看 `/api/v1/prompt-templates` 请求的状态

### 解决步骤

#### 步骤 1: 确认用户已登录
1. 访问 `http://localhost:3000/login`
2. 使用已有账号登录（如：`smoke@test.com`）
3. 确认登录后跳转到工作台页面

#### 步骤 2: 检查浏览器控制台
1. 访问 `http://localhost:3000/dashboard/prompt-templates`
2. 打开浏览器开发者工具（F12）
3. 查看 Console 标签页，是否有错误信息
4. 查看 Network 标签页，找到 `prompt-templates` 请求
   - 状态码应该是 `200`
   - 响应应该包含 `templates` 数组和 `total` 字段

#### 步骤 3: 验证后端日志
1. 查看后端服务日志
2. 当访问模板库页面时，应该能看到 API 请求日志
3. 如果有任何错误，会在日志中显示
